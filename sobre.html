# Luciano Júnior | Editor de Vídeo Site profissional desenvolvido para apresentação de serviços, portfólio e gerenciamento de conteúdo administrativo. ## Tecnologias * Frontend estático (HTML5, CSS3 e JavaScript) * GitHub Pages (Hospedagem) * Supabase (Banco de dados e Autenticação) * Supabase Auth (Login do painel administrativo) --- # Arquitetura ``` GitHub Pages │ ▼ Frontend (HTML/CSS/JavaScript) │ ▼ Supabase Auth │ ▼ Banco de Dados Supabase ``` O GitHub Pages hospeda todo o site. O Supabase é responsável por: * autenticação do administrador; * armazenamento dos dados; * comunicação via API; * gerenciamento das sessões de login. --- # Estrutura do Projeto ``` lucianosite-main/ │ ├── admin/ │ ├── login.html │ ├── index.html │ └── admin.js │ ├── assets/ │ ├── css/ │ ├── img/ │ └── js/ │ ├── components.js │ └── supabase-config.js │ ├── frontend/ │ ├── index.html │ ├── app.js │ ├── _headers │ └── _redirects │ ├── blog.html ├── contato.html └── README.md ``` --- # Funcionalidades * Página inicial * Página de contato * Blog * Painel administrativo * Login utilizando Supabase Auth * Integração com banco de dados Supabase * Componentes reutilizáveis em JavaScript --- # Configuração do Supabase A configuração está localizada em: ``` assets/js/supabase-config.js ``` Exemplo: ```javascript const SUPABASE_URL = "https://SEU-PROJETO.supabase.co"; const SUPABASE_ANON_KEY = "SEU_ANON_KEY"; ``` O projeto utiliza apenas a chave pública (Anon Key). **Nunca utilize a Service Role Key no frontend.** --- # Como executar localmente Basta utilizar qualquer servidor estático. Exemplo com VS Code: 1. Instale a extensão **Live Server**. 2. Clique em **Open with Live Server**. Ou utilize: ``` python -m http.server ``` Depois acesse: ``` http://localhost:8000 ``` --- # Publicação no GitHub Pages 1. Faça commit das alterações. ``` git add . git commit -m "Atualizações" git push ``` 2. No GitHub: Settings → Pages Configure: * Source: Deploy from a branch * Branch: main * Folder: / (root) Após alguns minutos o site estará publicado. --- # Painel Administrativo Página: ``` /admin/login.html ``` Após autenticação, o usuário é direcionado ao painel administrativo. O login utiliza o Supabase Auth. --- # Segurança O projeto utiliza: * Supabase Auth * Sessões autenticadas * Chave pública (Anon Key) * Comunicação HTTPS * Separação entre frontend e banco de dados Recomenda-se habilitar **Row Level Security (RLS)** em todas as tabelas do Supabase. --- # Tecnologias Utilizadas * HTML5 * CSS3 * JavaScript ES6 * Supabase * GitHub Pages --- # Fluxo da Aplicação ``` Usuário │ ▼ GitHub Pages │ ▼ JavaScript │ ▼ Supabase Auth │ ▼ Supabase Database ``` --- # Boas Práticas * Nunca exponha a Service Role Key. * Utilize RLS em todas as tabelas. * Faça backup periódico do banco de dados. * Utilize HTTPS. * Versione o projeto com Git. --- # Deploy Frontend * GitHub Pages Backend * Supabase Banco de Dados * PostgreSQL (Supabase) Autenticação * Supabase Auth --- # Licença Projeto de uso privado.
